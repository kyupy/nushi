# 院生室滞在ランキングアプリ 計画書

## 1. プロジェクト概要

院生室の在室時間を LIFF アプリで打刻し、Firestore に蓄積、各種ランキング・統計を可視化するWebアプリを構築する。

**コンセプト**
- メンバーのモチベ向上とネタ的な楽しみを目的とする
- 導入ハードルを下げるため LIFF（LINEログイン）を採用
- 将来の機能追加に耐える拡張性のあるデータ設計
- ネタアプリとはいえプライバシーにも配慮

**想定規模**
- ユーザー数: 10〜30人
- 打刻頻度: 1人あたり1日2〜6回程度
- 運用期間: 数年スパン（卒業生交代含む）

## 2. 技術スタック

| レイヤ | 技術 | 理由 |
|---|---|---|
| フロント | React + Vite + TypeScript | 開発体験・エコシステム・型安全性 |
| スタイル | Tailwind CSS | 素早く整った見た目 |
| 認証 | LIFF + Firebase Auth (Custom Token) | LINEログイン統合 |
| ホスティング | Firebase Hosting | LIFFエンドポイント、デプロイが楽 |
| DB | Cloud Firestore | リアルタイム、セキュリティルール、無料枠が広い |
| バックエンド | Cloud Functions for Firebase (2nd gen, TypeScript) | 集計バッチ、LINE API連携 |
| グラフ | Recharts + ECharts | 基本は Recharts、ヒートマップ等は ECharts |
| 通知 | LINE Messaging API | グループへのランキング投稿、個別通知 |
| 課金プラン | Blaze（外部API叩く必要があるため）※実質無料枠内 |

## 3. 機能要件

### 実装済み

- LIFFアプリでLINEログイン
- 入室/退室の打刻（前回状態から自動判定、ワンタップ完結）
- 打刻後に結果トーストを表示（`入室しました / 退室しました（滞在 3時間42分）`）
- 直近の打刻を数秒以内ならUIから取り消し可能（誤タップ救済）
- 自分の今月の滞在時間表示
- 月間総滞在時間ランキング
- 簡単なプロフィール画面
- プロフィール画面からオプトアウトトグル
- 打刻修正機能（履歴画面から日付・時刻・種類を指定して追加）
  - 修正可能な範囲: 過去1週間以内
  - 修正ログは `method: "manual-fix"` として記録
  - 追加打刻が最新ログの場合は `currentStatus` も更新
- 履歴画面の各ログ行にゴミ箱ボタン（任意のログを削除可能）
  - 削除時に対応セッションも無効化・統計再計算
- 退室忘れ通知（12:00 JST に6時間以上在室のユーザーへLINE個別通知）
  - Quick Reply: 「今すぐチェックアウト」「まだ在室中」
- 毎月1日 09:00 JST に前月ランキングをLINEグループ投稿（LINE_GROUP_ID設定済み・未テスト）
- LINE Bot Webhook（Quick Replyのpostback処理・動作確認済み）

### Phase 2

- 複数種類のランキング（早起き・夜更かし・皆勤・週末戦士など）
- 曜日×時間帯のヒートマップ
- 累積滞在時間の折れ線グラフ
- 個人の「タイプ診断」レーダーチャート

### Phase 3 以降

- 同時在室ネットワーク（誰と一緒にいがちか）
- 年間MVP、各種アワード
- 管理者ダッシュボード

## 4. データ設計（Firestore）

### コレクション構造

```
users/{userId}
logs/{logId}                      生ログ（不変・監査用）
sessions/{sessionId}              入退室ペア
stats/daily/records/{date}        日次集計
stats/monthly/{yearMonth}/_root_  月次集計（ランキング含む）
stats/monthly/{yearMonth}/{userId} 個人詳細統計
config/                           アプリ設定、管理者リスト等
```

### users/{userId}

LINEのuserIdをドキュメントIDとする。

```typescript
{
  userId: string;              // LINE userId
  displayName: string;         // 現在の表示名
  pictureUrl: string | null;
  joinedAt: Timestamp;
  currentStatus: "in" | "out";
  lastActionAt: Timestamp;
  currentSessionId: string | null;
  optOut: boolean;             // ランキング非表示
  role: "member" | "admin";
  schemaVersion: number;
}
```

### logs/{logId}

絶対に改変しない生ログ。論理削除のみ許可。

```typescript
{
  userId: string;
  displayName: string;
  action: "in" | "out";
  timestamp: Timestamp;
  clientTimestamp: Timestamp;
  date: string;                // "2026-04-08"（05:00境界）
  yearMonth: string;           // "2026-04"
  year: number;
  dayOfWeek: number;           // 0=日
  hour: number;
  isWeekend: boolean;
  isHoliday: boolean;
  platform: "ios" | "android" | "pc" | "other";
  appVersion: string;
  liffVersion: string;
  method: "button" | "auto-close" | "manual-fix" | "undo";
  voided: boolean;
  voidedAt: Timestamp | null;
  raw: Record<string, unknown>;
  schemaVersion: number;
}
```

### sessions/{sessionId}

退室時に Cloud Functions が自動生成する入退室ペア。

```typescript
{
  userId: string;
  displayName: string;
  checkIn: Timestamp;
  checkOut: Timestamp;
  durationSeconds: number;
  date: string;
  yearMonth: string;
  dayOfWeek: number;
  hourIn: number;
  hourOut: number;
  isWeekend: boolean;
  isHoliday: boolean;
  nightSeconds: number;        // 22時〜翌5時の秒数
  coreSeconds: number;         // 平日9-18時の秒数
  autoClosed: boolean;
  voided: boolean;
  manualFix?: boolean;         // fixStamp が直接生成したセッション（onSessionCreate をスキップ）
  schemaVersion: number;
}
```

### stats/monthly/{yearMonth}/_root_

ランキング画面は親ドキュメント1回読むだけで済むようにする。

```typescript

      totalSeconds: number;

    daysPresent: string[];

    nightOwl: string[];
    weekendWarrior: string[];
    streak: string[];
  };
}
```

### stats/monthly/{yearMonth}/{userId}

個人詳細統計。

```typescript

  userId: string;
  displayName: string;
  earliestIn: string;          // "07:32"

```

## 5. 処理フロー

### 打刻フロー

```
1. ユーザーがドアのQRを読む
2. LIFFアプリ起動 → liff.init() → liff.getIDToken()
3. callable 関数 authWithLine に ID トークンを送信
4. Functions 側で LINE API を使い ID トークンを検証
5. 検証成功 → Firebase Custom Token 発行
6. フロントで signInWithCustomToken でサインイン
7. ホーム画面に大きな打刻ボタン1つ
8. タップ → callable checkIn を呼ぶ（確認画面なし）
9. 成功したらトースト表示
10. 直後の10秒間「取り消し」ボタンを表示
11. onLogCreate トリガー発火:
    - action が "out" かつ method が "manual-fix" でなければ session 生成
    - 直前の "in" ログとペアリング
12. onSessionCreate で stats/monthly を増分更新
    - manualFix: true のセッションはスキップ（fixStamp が直接再計算するため）
```

### 誤タップ救済の仕組み

- ホーム画面に「直前の打刻を取り消す」ボタンを10秒間表示
- タップすると直近のログを論理削除（`voided: true`）
- 対応する session も無効化・統計再計算
- 10秒経過後は「履歴」画面のゴミ箱ボタンで個別削除が可能

### 打刻修正フロー

```
1. 履歴画面で「打刻を追加・修正」をタップ
2. 日付・時刻（15分刻み）・種類（入室/退室）を選択
3. callable 関数 fixStamp を呼ぶ
4. バリデーション:
   - 1週間以内であること
   - 未来でないこと（バックエンドで弾く）
   - 入退室の順序が矛盾しないこと
5. logs に method: "manual-fix" で記録
6. 追加打刻が最新ログなら currentStatus を更新
7. 該当日のセッションを全件void → ログから再生成（manualFix: true で作成）
8. stats/monthly と stats/daily を再計算
```

### ログ削除フロー

```
1. 履歴画面のゴミ箱ボタンをタップ
2. callable 関数 voidLog を呼ぶ
3. ログを voided: true に
4. 「out」なら checkOut が一致するセッションを void
   「in」なら checkIn が一致するセッションを void
5. stats/monthly と stats/daily を再計算
6. currentStatus を直前のログの状態に巻き戻す
```

### 集計フロー

**日付境界**: 05:00 JST（院生の深夜利用に対応）

| タイミング | 処理 |
|---|---|
| logs 追加時 | sessions 生成（manual-fix は除く） |
| sessions 追加時 | stats/monthly 増分更新（manualFix は除く） |
| 毎日 05:30 JST | stats/daily 確定、min/max 再計算、12時間超を自動クローズ |
| 毎日 12:00 JST | 6時間以上在室中のユーザーに LINE 個別通知 |
| 毎月 1日 09:00 JST | 前月ランキングを LINE グループ投稿 |
| 手動 | 全期間再集計 callable |


## 7. プライバシー方針

- **位置情報は取得しない**
- **生IPは保存しない**
- **LINEアクセストークンは検証のみ、保存しない**
- **退会・卒業時**: 管理者操作で匿名化
- **オプトアウト**: プロフィール画面からトグル、ランキングから除外
- **ログ保存期間**: 無期限
- **公開範囲**: ログイン済みメンバーのみ

## 11. 備考

- 生ログは**絶対に消さない・変えない**
- `schemaVersion` をつけておくことで後日マイグレーションが書きやすくなる
- 日付境界は 05:00 JST（`logicalDate` / `logicalYearMonth` 関数で管理）
- `stats/daily` のパスは `stats/daily/records/{date}`（4セグメント）。3セグメントはFirestoreで無効

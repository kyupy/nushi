# 院生室滞在ランキングアプリ 計画書

## 1. プロジェクト概要

院生室の在室時間を LIFF アプリで打刻し、Firestore に蓄積、各種ランキング・統計を可視化するWebアプリを構築する。

**コンセプト**
- メンバーのモチベ向上とネタ的な楽しみを目的とする
- 導入ハードルを下げるため LIFF（LINEログイン）を採用
- 将来の機能追加に耐える拡張性のあるデータ設計
- ネタアプリとはいえプライバシーにも配慮

**想定規模**
- ユーザー数: 10〜30人
- 打刻頻度: 1人あたり1日2〜6回程度
- 運用期間: 数年スパン（卒業生交代含む）

---

## 2. 技術スタック

| レイヤ | 技術 | 理由 |
|---|---|---|
| フロント | React + Vite + TypeScript | 開発体験・エコシステム・型安全性 |
| スタイル | Tailwind CSS | 素早く整った見た目 |
| 認証 | LIFF + Firebase Auth (Custom Token) | LINEログイン統合 |
| ホスティング | Firebase Hosting | LIFFエンドポイント、デプロイが楽 |
| DB | Cloud Firestore | リアルタイム、セキュリティルール、無料枠が広い |
| バックエンド | Cloud Functions for Firebase (2nd gen, TypeScript) | 集計バッチ、LINE API連携 |
| グラフ | Recharts + ECharts | 基本は Recharts、ヒートマップ等は ECharts |
| 通知 | LINE Messaging API | グループへのランキング投稿、個別通知 |
| 課金プラン | Blaze（外部API叩く必要があるため）※実質無料枠内 |

---

## 3. 機能要件

### MVP（最初にリリースする範囲）

- LIFFアプリでLINEログイン
- 入室/退室の打刻（前回状態から自動判定、ワンタップ完結）
- 打刻後に結果トーストを表示（`入室しました / 退室しました（滞在 3時間42分）`）
- 直近の打刻を数秒以内ならUIから取り消し可能（誤タップ救済）
- 自分の今月の滞在時間表示
- 月間総滞在時間ランキング
- 簡単なプロフィール画面
- プロフィール画面からオプトアウトトグル（ランキング非表示にできる。メンバーの心理的安全性のためMVPから提供）
- 打刻修正機能（入室忘れ・退室忘れの自己修正）
  - LIFF アプリ内「履歴」画面から修正（日付・時刻・種類を指定して入室/退室を後から記録）
  - 修正可能な範囲: 過去1週間以内（それ以降は管理者対応）
  - 修正ログは `method: "manual-fix"` として記録し、通常打刻と区別
- LINE Bot 経由の簡易修正（退室忘れ通知から直接修正可能）
  - スケジュール関数が「6時間以上在室中」を検知 → LINE に個別通知
  - 通知に Quick Reply ボタン: 「今退室する」「○時に退室した」「まだ在室中」
  - 「○時に退室した」を選ぶと LIFF の修正画面に遷移（時刻プリセット付き）

### Phase 2

- 複数種類のランキング（早起き・夜更かし・皆勤・週末戦士など）
- 曜日×時間帯のヒートマップ
- 累積滞在時間の折れ線グラフ
- 個人の「タイプ診断」レーダーチャート
- 月初の自動ランキング投稿（LINEグループ）

### Phase 3 以降

- 同時在室ネットワーク（誰と一緒にいがちか）
- 年間MVP、各種アワード
- 管理者ダッシュボード

## 4. データ設計（Firestore）

### コレクション構造

```
users/{userId}
logs/{logId}                生ログ（不変・監査用）
sessions/{sessionId}        入退室ペア
stats/daily/{date}          日次集計
stats/monthly/{yearMonth}   月次集計（ランキング含む）
stats/allTime/summary       全期間集計
config/                     アプリ設定、管理者リスト等
```

### users/{userId}

LINEのuserIdをドキュメントIDとする。

```typescript
{
  userId: string;              // LINE userId
  displayName: string;         // 現在の表示名
  pictureUrl: string | null;
  joinedAt: Timestamp;
  currentStatus: "in" | "out";
  lastActionAt: Timestamp;
  currentSessionId: string | null;
  optOut: boolean;             // ランキング非表示
  role: "member" | "admin";
  schemaVersion: number;
}
```

### logs/{logId}

絶対に改変しない生ログ。スキーマ変更時は `schemaVersion` を上げる。

```typescript
{
  userId: string;
  displayName: string;         // スナップショット
  action: "in" | "out";
  timestamp: Timestamp;        // サーバー時刻
  clientTimestamp: Timestamp;  // クライアント申告時刻

  // 時間分解（クエリ効率化のため冗長に持つ）
  date: string;                // "2026-04-08"
  yearMonth: string;           // "2026-04"
  year: number;
  dayOfWeek: number;           // 0=日
  hour: number;
  isWeekend: boolean;
  isHoliday: boolean;

  // 環境
  platform: "ios" | "android" | "pc" | "other";
  appVersion: string;
  liffVersion: string;

  // 打刻方法
  method: "button" | "auto-close" | "manual-fix" | "undo";

  // 論理削除（誤タップ取り消し用）
  voided: boolean;
  voidedAt: Timestamp | null;

  // 拡張用
  raw: Record<string, unknown>;
  schemaVersion: number;
}
```

### sessions/{sessionId}

退室時に Cloud Functions が自動生成する入退室ペア。

```typescript
{
  userId: string;
  displayName: string;
  checkIn: Timestamp;
  checkOut: Timestamp;
  durationSeconds: number;

  date: string;
  yearMonth: string;
  dayOfWeek: number;
  hourIn: number;
  hourOut: number;
  isWeekend: boolean;
  isHoliday: boolean;

  nightSeconds: number;        // 22時〜翌5時の秒数
  coreSeconds: number;         // 平日9-18時の秒数

  autoClosed: boolean;         // 退室忘れを自動クローズしたか
  schemaVersion: number;
}
```

### stats/monthly/{yearMonth}

ランキング画面は親ドキュメント1回読むだけで済むようにする。個人の詳細統計はサブコレクションに分離し、ドキュメント肥大化を防ぐ。

**親ドキュメント: stats/monthly/{yearMonth}**（ランキング + 基本指標）

```typescript
{
  yearMonth: string;
  updatedAt: Timestamp;

  users: {
    [userId: string]: {
      displayName: string;
      pictureUrl: string | null;
      totalSeconds: number;
      daysPresent: number;
      sessionCount: number;
      avgSessionMinutes: number;
    };
  };

  rankings: {
    totalTime: string[];       // userIdの配列（順位順）
    daysPresent: string[];
    earlyBird: string[];
    nightOwl: string[];
    weekendWarrior: string[];
    streak: string[];
  };
}
```

**サブドキュメント: stats/monthly/{yearMonth}/users/{userId}**（個人詳細統計）

```typescript
{
  userId: string;
  displayName: string;
  earliestIn: string;          // "07:32"
  latestOut: string;           // "23:45"
  weekendSeconds: number;
  nightSeconds: number;
  coreSeconds: number;
  maxStreak: number;           // 最長連続日数
  heatmap: number[][];         // [7][24] 曜日×時間帯の滞在秒数
  schemaVersion: number;
}
```

ランキング画面は親ドキュメントのみ、個人統計画面は親 + サブドキュメント1件を読む。

## 5. 処理フロー

### 打刻フロー

```
1. ユーザーがドアのQRを読む
2. LIFFアプリが起動、liff.init() → liff.getProfile()
3. liff.getIDToken() で LINE ID トークンを取得
4. callable 関数 authWithLine に ID トークンを送信
5. Functions 側で LINE API を使い ID トークンを検証
6. 検証成功 → Firebase Custom Token を発行して返却
7. フロントで signInWithCustomToken でサインイン
8. users/{userId} から currentStatus を取得
9. ホーム画面に大きな打刻ボタン1つ
   （在室中なら「退室する」、不在なら「入室する」と表示）
10. タップ即座に callable 関数を呼ぶ（確認画面なし）
11. 成功したらトースト表示
    例: 「退室しました。お疲れさまでした（滞在 3時間42分）」
12. 直後の数秒間「取り消し」ボタンを表示（誤タップ救済）
13. onLogCreate トリガーが発火:
    - action が "out" なら sessions を生成
    - 直前の "in" ログとペアリング
    - 異常値（12h超など）検知
14. onSessionCreate で stats/monthly を増分更新
```

### 誤タップ救済の仕組み

- ホーム画面に「直前の打刻を取り消す」ボタンを10秒程度表示
- タップすると直近のログを論理削除（`voided: true` フラグを立てる、物理削除はしない）
- Cloud Functions 側で対応する session も無効化
- 10秒経過後は「履歴」画面から自分で打刻修正が可能（過去1週間以内）

### 打刻修正フロー（入室忘れ・退室忘れの自己修正）

```
1. 履歴画面で「打刻を追加・修正」をタップ
2. 日付・時刻・種類（入室/退室）を選択
   - デフォルトは「現在のステータスの逆」
   - 時刻は15分刻みピッカー（手入力も可）
3. callable 関数 fixStamp を呼ぶ
4. バリデーション:
   - 1週間以内であること
   - 入退室の順序が矛盾しないこと（入室→退室→入室→…）
   - 重複セッションが発生しないこと
5. logs に method: "manual-fix" で記録
6. 必要に応じて既存 sessions を再生成
7. stats/monthly を再計算（該当月のみ）
```

**LINE Bot 経由の修正フロー**（退室忘れ通知からの修正）:

```
1. スケジュール関数が「6時間以上在室中」のユーザーを検知
2. LINE Messaging API で個別通知を送信
3. Quick Reply ボタン:
   - 「今退室する」→ callable checkIn を呼んで即時退室処理
   - 「○時に退室した」→ LIFF の修正画面に遷移（時刻プリセット付き）
   - 「まだ在室中」→ 何もしない（次回通知は翌日）
4. 「○時に退室した」の場合: LIFF 修正画面で時刻を確認・調整して fixStamp を実行
```

### ネットワークエラー対応

- LIFF起動時にネットワーク接続を確認し、不安定なら警告バナーを表示
- 打刻ボタンのタップ後にタイムアウト（5秒）で「接続できませんでした。再試行してください」を表示
- オフラインキューは設けない（複雑になるため割り切り。後から打刻修正機能で補完可能）

### 集計フロー

**日付境界の定義**: 1日の境界は **05:00 JST**（05:00〜翌04:59を1日とする）。院生は深夜帯の利用がありうるため、0時をまたぐセッションを自然に1日に収める。

| タイミング | 処理 | 種類 |
|---|---|---|
| logs 追加時 | sessions 生成 | Firestore trigger |
| sessions 追加時 | stats/monthly の数値を増分更新 | Firestore trigger |
| 毎日 05:30 JST | 前日（05:00〜04:59）分の stats/daily 確定、min/max 系再計算 | Scheduled |
| 毎日 05:30 JST | 12時間以上在室中のユーザーを自動クローズ（後述） | Scheduled |
| 毎日 12:00 JST | 6時間以上在室中のユーザーに LINE 通知（Quick Reply で修正可能） | Scheduled |
| 毎月 1日 09:00 JST | 前月ランキングをLINEグループ投稿 | Scheduled |
| 手動 | 全期間再集計コマンド（バグ修正時） | HTTPS callable |

### 退室忘れ自動クローズ

- **自動クローズ閾値: 12時間**（config コレクションに格納、運用に応じて調整可能）
- 自動クローズ時: 入室から12時間の時点を退室時刻として session を生成（`autoClosed: true`）
- ユーザーに LINE 通知を送り、正しい退室時刻への修正を促す（打刻修正機能で修正可能）
- 修正されたら自動クローズの session を無効化し、正しい session に置き換え

### 集計戦略

- **単純な合計系**（totalSeconds, sessionCount 等）: 増分更新でリアルタイム反映
- **min/max・順位・streak 系**: 日次バッチで再計算
- 当月途中のランキングには「◯日 05:30 時点」の注記をつける

## 6. セキュリティルール方針

```
users/{userId}
  read:  本人 or 管理者 or (他人かつ optOut=false の公開フィールドのみ)
  write: Functions のみ

logs/{logId}
  read:  本人 or 管理者
  write: Functions のみ（クライアントは callable 経由で打刻）

sessions/{sessionId}
  read:  本人 or 管理者
  write: Functions のみ

stats/**
  read:  認証済みユーザー全員
  write: Functions のみ

config/**
  read:  認証済みユーザー
  write: 管理者のみ
```

クライアントからの直接書き込みは原則禁止し、すべて Cloud Functions の callable 関数経由とする。

## 7. プライバシー方針

- **位置情報は取得しない**（LIFFの権限リクエストも出さない）
- **生IPは保存しない**（必要ならハッシュ化）
- **LINEアクセストークンは検証のみ、保存しない**
- **退会・卒業時**: 管理者操作で匿名化（displayName を "卒業生A" 等に置換、userIdはそのまま）
- **オプトアウト**: プロフィール画面からトグル、ランキングから除外される
- **ログ保存期間**: 無期限（容量に応じて見直し）
- **公開範囲**: ログイン済みメンバーのみ。ランキングは院生室メンバー内で共有
